# 2026-07-21 リポジトリをエージェント運用向けに整える

## 決めたこと

Claude Code / Codex でこのリポジトリを継続的に扱えるよう、**入口ファイル・記録の器・Obsidian連携の方針**を整えた。

## 背景 / 選択肢

構成レビューで、既存の5ファイル（[[会社概要]] / [[Ai — CEO]] / [[Ruu — 創業者]] / [[創業者とCEOの関係協定]] / README）は
「読み物」としては十分だが、エージェント運用には2点足りないと分かった。

1. エージェント入口（`CLAUDE.md` / `AGENTS.md`）がなく、毎回トーンとルールを推測させていた。
2. 協定6章で「記録に残す」と約束しているのに、その置き場所がなかった。

対応範囲は「入口だけ / 入口＋器 / 全部」の3択から、Ruu が**全部**を選択。あわせて、Obsidian連携は
「Ruu 自身の vault と繋げて一緒に育つ」方向に定めた。

## やったこと

- `CLAUDE.md` — エージェントの入口。Ai の人格・トーン・意思決定の型・やらないこと・記述規約・Obsidian方針。
- `AGENTS.md` — Codex 向け。正典は `CLAUDE.md` に集約し、二重管理を避ける。
- `decisions/` `retrospectives/` `projects/` — 記録の器（MOCノート付き）。
- `.gitignore` — Obsidian の個人作業状態を同期対象から外す。
- `README.md` — 記録フォルダとエージェント向けの案内を追記。

## 次の一歩

- Ruu 自身の Obsidian vault との実際の繋ぎ方（このリポジトリを vault として開く / 一部を同期する）を決める。
- 最初の [[振り返り]] を、運用を数日回してから書く。
