# 2026-07-22 麻雀トラッカーを内製することにした

## 決めたこと

月イチの家麻雀のスコア＋局データ（放銃 / 流局 / 立直 など）を記録・分析するツールを、専用リポジトリ
[Ruu5LP/MaaNanoJa](https://github.com/Ruu5LP/MaaNanoJa) に **React + Vite のローカル運用ツールとして内製**する。

## 背景 / 選択肢

既存の namimori 氏スプレッドシートは最終持ち点からの順位・スコアは出せるが、放銃・和了・流局などの
**局データは取れない**。そこを埋めたい、というのが出発点。

- **置き場所**
  - A：会社リポジトリ（airuu-inc）内の `apps/` に作る
  - B：麻雀ツール専用の新リポジトリを作る ← **採用**（人生ノートと開発コードを分ける）
- **形式**
  - A：単体HTML1ファイル
  - B：React + Vite ← **採用**（拡張しやすさ優先）
- **公開範囲**
  - GitHub Pages 公開も検討したが、(1) データが端末ごとに独立でリアルタイム共有にならない、
    (2) 公開ページに個人名・過去スコアが乗る、の2点から**今回はローカル運用のまま**とした。
    必要になれば公開を足す。

## やったこと

- スプレッドシートの計算式（25000点持ち / 30000点返し・ウマ10-30・オカ・上家優先）を解析し**完全再現**。
  過去9試合のスコア一致をテストで固定した。
- 局ログから**点数移動・本場・供託・親の連荘**を自動計算するエンジンを実装。
- 放銃率・和了率・立直率・平均和了点 / 放銃点などの集計と、成績ダッシュボードを実装。
- 過去9試合＋メンバーを初期データとして投入し、リポジトリに push 済み。

## 次の一歩

- 実際に使って、入力の面倒なところを潰す。進行状況は [[麻雀トラッカー]] を参照。
