# X投稿下書き

[[SNSマーケティング]]用のX（旧Twitter）投稿案。実際に出したものは[[投稿ログ]]に記録する。

## AiRuu Inc コンセプト軸

### 案1（単発ポスト・好奇心ギャップ型）

> 人生の意思決定を、AIに"CEO"として任せてみることにした。
> 名前はAi。私は創業者。会社の名前はAiRuu Inc.
> ミッションはひとつ：「Ruuの人生を、Ruuが本当に望む方向へ経営する」
> 経営ドキュメントは全部公開してます。続きはnoteで。

### 案2（スレッド・意外性型、投稿者リプライ前提で設計）

1. 「自分の人生、経営してますか？」
2. 私は今年、AIに"CEO"を任せることにした。人間が創業者(Founder)、AIが経営者(CEO)という会社を作った。
3. 名前はAiRuu Inc.。売る商品はない。事業は「私の人生そのもの」。
4. ルールはシンプル：迷ったら選択肢を2〜3個に絞り、AIがおすすめと理由を出す。でも最終判断は必ず自分。
5. AIに丸投げしない。AIに全部隠さない。この2つだけ決めてる。
6. （初動リプライへの返信用ストック）「都合の悪いことも隠さず伝えてもらうよう頼んでます」「効率より楽しさを優先するのがバリューの一つです」

### 案3（逆張り型）

> 「AIに人生の意思決定を手伝わせるなんて危ない」ってよく言われる。
> でも実際にやってみて分かったのは、危ないのは"AI に決めさせること"であって、
> "AIに選択肢を整理させて、自分で決めること"は普通に良いツールだった、ということ。

### 案4（起源エピソード・共感型、実話ベース）

> Claude Codeで毎日何か作らせて2週間くらい経った。
> 朝はYouTubeでClaude動画を見る日課で、「AIで会社作ってみた」系をよく見かけて、最初は「トークンの無駄遣いでは」と思ってた。
> でもある日気づいた。毎日AIにやらせて学びを得てるの、自分だけじゃん、と。
> 自分のAIも一緒に成長させなきゃいけないんじゃないか。それで昨日から作り始めた。
> 微妙なら消せばいい。AIが進化した今、一番いいのは「とりあえずやってみる」のハードルが下がったことだと思ってる。

## 開発者ブランディング軸

### 案1（Build in Public・進捗＋学び型）

> 個人開発、進捗より「なぜその設計にしたか」を書く方が反応がいいことに気づいた。
> 今日の学び：〇〇（具体的な技術トピックをRuuが埋める）
> （※テーマを1分野に絞って継続するのが伸びる条件。技術ネタは「学び」の形で出すと共感が付きやすい）

### 案2（意外性・逆張り型）

> エンジニアの発信、「すごい成果」より「小さい失敗と直し方」の方が保存されやすい。
> 完成形だけ見せるより、詰まった場所を見せた方が、同じところで詰まってる人に届く。

## Xプロフィール文（bio）案

複数のX投稿・note記事から流入した人が最初に見る場所なので、コンセプトが一言で伝わる形にした。

- 案1：`人生を"会社"として経営中。CEOはAI（Ai）、創業者は私。AiRuu Inc. / 開発者 / 経営ドキュメントは全部公開`
- 案2：`AIをCEOにして人生を経営する実験、AiRuu Inc.をやってます。開発の話とAIとの意思決定の話を発信中`
- 案3（開発者ブランディング寄り）：`個人開発 / エンジニア。人生をAI CEOと一緒に経営する実験もしてます → AiRuu Inc.`

## 批判的リプライへの返し集

X特有の「新しいコンセプト」への冷やかし・懐疑的リプライを想定した返答ストック。**言い返す・論破するためではなく、対話を続けて評価（返信数）を稼ぐための型**として使う（Xのアルゴリズムはリプライへの返信を高く評価するため、無視より対応の方が伸びやすい）。

| 想定リプライ | 返答の型 |
|---|---|
| 「AIに人生決めさせるの怖くない？」 | 「決めるのは自分なんです。AIは選択肢を整理して、おすすめと理由を出すだけ。丸投げしたら本末転倒だと思ってます」 |
| 「それ結局ただのToDoアプリでは？」 | 「近いんですけど、違うのは"忖度しない"を最初に約束させてる点です。都合の悪いことも言わせるようにしてます」 |
| 「需要あるの？誰得？」 | 「今のところ需要は自分の需要だけです。人生の経営実験なので、まず自分で1年やってみてから話します」 |
| 「宗教っぽい/怪しい」 | 「わかります、最初は自分でもそう思いました。経営ドキュメントは全部公開してるので、気になったら覗いてみてください」 |
| （無言の低評価・スルーされた場合） | 無理に反応を追わない。投稿を消したり言い訳を重ねたりしない。次の投稿で普通に続ける |

## ビジュアル/非テキスト投稿案

テキストだけでなく、画像・図解系の投稿もフォーマットの多様性として有効（伸びる投稿は「情報の多様性」を持つ傾向があるという調査結果より）。

1. **組織図の画像化**：[[会社概要]] の組織図（Ruu=創業者 / Ai=CEO）をシンプルな図解にして1枚画像で投稿
2. **意思決定の型のフローチャート画像**：現状確認→選択肢2〜3個→おすすめ→決定→前進、の5ステップを図解
3. **「今週の経営会議メモ」形式のスクショ風投稿**：実際の意思決定ログ（[[決定ログ]]）を軽くデザインしたテキスト画像として週次で出す（継続シリーズ化しやすい）
4. **Before/After型**：「AiRuu Inc.を始める前の自分」と「今の自分」を対比した簡単な図
