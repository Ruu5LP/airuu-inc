# 麻雀トラッカー

月イチの家麻雀（4人打ち）のスコアと、放銃・流局・立直などの局データを記録して、そこから成績を分析するためのローカルツール。Ruuの「好きなこと（＝麻雀）」まわりの開発案件。

## 目的

- いつも使っている namimori 氏の麻雀集計スプレッドシートの計算を**そのまま再現**しつつ、
- スプレッドシートでは取れなかった**局ごとの記録（和了 / 放銃 / 流局 / 立直）→ 放銃率などのデータ**を取れるようにする。

## 状態

**v1 完成・稼働中。** 専用リポジトリ [Ruu5LP/MaaNanoJa](https://github.com/Ruu5LP/MaaNanoJa)（private）に構築済み。
今回は**ローカル運用**（GitHub Pages 公開は見送り、必要になれば追加）。

- スコア計算は既存スプレッドシートの**過去9試合と完全一致**することをテストで確認済み。
- 過去9試合＋メンバー（れいに / もっそ / さーりゃん / 超髪）を初期データとして投入済み。開いた瞬間から成績が見える。
- **設計ルールを整備して main にマージ済み**（TypeScript strict 化・ESLint/Prettier・Vitest・GitHub Actions でゲート化・`CLAUDE.md`/`SPEC.md`）。
  「誰が入っても同じ品質」「AIが一番読みやすい状態」を土台にした。経緯は [[フロントの設計ルールを型とCIで縛ることにした]]。

## 主な機能

- **記録**：局ログ入力（点数移動・本場・供託リーチ棒・親の連荘まで自動計算）／最終点だけ入力の2モード。
- **成績**：合計スコア・平均順位・着順分布・トップ / 連対 / ラス / トビ率、放銃率・和了率・立直率・平均和了点 / 放銃点 など。
- **保存**：localStorage、JSON 書き出し / 読み込みでバックアップ。
- **技術**：React + Vite、バックエンドなし（ブラウザ内で完結）。

## 次の一歩

- まず実際に一度使ってみて、**入力の面倒なところ**を洗い出す（一番効く改善はここから）。
- 欲しくなったら検討：入力の簡略化（役選択・点数の直接入力）／ GitHub Pages での配布／全員同時入力（＝要バックエンド）。
